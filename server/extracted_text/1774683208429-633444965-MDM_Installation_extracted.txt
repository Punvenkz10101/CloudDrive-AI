Installation Overview
Component
Description
MDM Hub Master Database
A schema that stores and consolidates business data for 
the MDM
 Hub. Contains environment configuration settings such as user accounts, security configuration, Operational Reference Store registry, and message queue settings. You can access and manage Operational Reference Stores from the Master Database. Default name is CMX_SYSTEM, but a custom name can be used. Multiple Master Databases can exist in the same database instance.
Operational Reference Store (ORS)
A schema that stores and consolidates business data for 
the MDM
 Hub. Contains master data, content metadata, and rules to process and manage the master data. Separate ORS databases can be configured for different geographies, departments, or environments (dev/prod). ORS databases can also be distributed across multiple servers. Default name is CMX_ORS.
Hub Server
A
 application deployed on an application server. Processes data stored within the MDM Hub and 
integrates
 with external applications. Manages core and common services for 
the MDM
 Hub.
Process Server
A
 application deployed on an application server. Processes batch jobs such as load
 
and revalidate. Performs data cleansing and match operations. Interfaces with the cleanse engine to standardize and optimize data for match and consolidation.
Informatica 
ActiveVOS
®
A BPM tool required internally by 
the MDM
 Hub for processing data. Supports automated business processes including change-approval workflows. Ensures master data changes go through review and approval before inclusion in records. When installed with Hub Server, it installs 
ActiveVOS
 Server, 
ActiveVOS
 Console, Process Central, and predefined MDM workflows, tasks, and roles.
